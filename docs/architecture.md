# 架构说明

这个项目的核心不是单脚本，而是一条分层工作流。每一层只解决一个问题，避免“一个脚本包办全部”。

## 分层

1. 采集层：从 ChinaMoney 或其他官方来源拿到原始材料。
2. 解析层：用 MinerU 把 PDF、Word、PPT 或图片转成可读中间文本。
3. 标准化层：识别主体、期间、附注范围和章节结构。
4. 分析层：按附注优先原则分析主体的财务结构、债务压力、现金覆盖、受限资金、担保、或有事项和条款风险，并生成工作底稿和报告。
5. 导出层：把稳定结果放进对外交付格式。
6. 运行治理层：记录 manifest、状态、重跑和失败原因。
7. 知识治理层：管理正式知识库与回滚。

## 为什么要这样拆

- 采集和解析经常失败，但失败原因不同。
- 标准化和分析需要不同的人读不同的中间产物。
- 对外交付内容不能和内部知识演进混在一起。
- 后续团队协作时，边界越清楚，越不容易互相污染。

## 三个 skill 的角色

- `chinamoney` 负责拿材料，确保财报、评级报告和披露文件可获取。
- `mineru` 负责把材料读出来，把非结构化文档变成可分析的中间文本。
- `financial-analyzer` 负责把读出来的材料变成债券发行主体的财务分析结果，重点关注附注、债务、现金、条款和风险信号。

## 文档边界

- 项目内只保留开发与协作必须的文档。
- 长篇业务说明不要放进仓库主干文档里。
