# 快速上手

这是一份最短路径说明。目标不是一次性学完全部项目，而是先知道项目结构、关键入口和最小运行条件。

## 你会先接触到什么

- `financial-analyzer`：债券发行主体的财务分析技能，负责把财报和附注转成工作底稿、报告和结构化结果。
- `chinamoney`：财报与披露下载。
- `mineru`：文档解析。

## 最小理解路径

1. 先看 [README.md](../README.md)，确认项目入口和目录结构。
2. 再看 [architecture.md](architecture.md)，理解采集、解析、分析、导出的层级。
3. 然后看三个 `SKILL.md`，知道各自负责什么，不要把它们混成一个脚本。
4. 最后看 [contracts.md](contracts.md)，确认关键 JSON/Markdown 产物的名字和用途。

## 最小运行条件

- Python 环境可用。
- 能读取仓库内的三个 skill。
- 有可用的原始 PDF 或已解析 Markdown。
- 如果要跑解析，`MINERU_TOKEN` 需要可用。

## 先不要做的事

- 不要先改分析逻辑再补文档。
- 不要把 scaffold 误认为最终报告。
- 不要把运行态目录写回 skill 目录。

## 一次最小案例的目标

你至少要能说清楚以下四件事：

- 这个案例是哪家主体、哪一年、哪类报告。
- 原始文件从哪里来。
- 解析后中间产物放在哪里。
- 哪些文件是草稿，哪些文件才算正式产物。

如果你已经理解这四件事，再往下看就应该能顺着分析路径找到附注、债务、现金、受限资金和条款信息，而不是只停留在报表摘要层面。
